## САММАРИ (для CEO – одна страница)

### Проблема
- Выручка 267 млн ₽ за 6 лет, но 56% от одного врача (Мадякин).
- Удержание упало с 65% (2022) до 55% (2025) – каждый второй новый пациент не возвращается.
- 1 089 «китов» с историей 109 млн ₽ лежат без дела (сегменты RiskWhale, SleepingWhale, LostWhale).
- 1С (БИТ.УМЦ) не даёт гибкой автоматизации, нет API, нет удобной маркетинговой сегментации.
- Текущий маркетинг: 195 тыс. ₽/мес дают хаотичный контент и слабые VK Ads. Контент местами нарушает этику (пациенты в белье, дети без одежды).

### Решение – STRONG AIOS CRM
- Полный отказ от 1С. Все данные мигрируются один раз.
- Кастомная CRM на **Python (FastAPI) + PostgreSQL + n8n + RabbitMQ**.
- **Голосовой AI-бот «Служба заботы»** с клонированным голосом главврача (24/7):
  - Подтверждение записей → снижение no-show на 30–50%.
  - Возврат не пришедших (звонок в течение часа).
  - Реактивация «китов» (1 089 пациентов) по триггерам.
  - Follow-up после первого приёма (конверсия в курс).
  - Сбор отзывов (Яндекс.Карты, 2ГИС, ProDoctors).
  - Поздравления с днём рождения, акции.
- Автоматическая RFM-сегментация, еженедельные кампании обзвона.
- Воронка для 4 ЦА: магнит → квиз → tripwire → курс.
- Интеграция с Avito Ads, ProDoctors, Telegram, MAX.

### Эффект (прогноз на 6 месяцев)
- Возврат минимум 15% из 1 089 «китов» → ~160 пациентов × средний чек 100K = **16 млн ₽ доп. выручки**.
- Снижение no-show на 30% → +5–8 млн ₽ от сохранённых записей.
- Повышение конверсии 1-й визит → курс с 25% до 40% → +~300 курсов в год.
- Окупаемость разработки (1,2 млн ₽) за 4–6 месяцев.

## 2. ДЕТАЛЬНАЯ АРХИТЕКТУРА (схема + описание потоков)

```
Пользователи (админ, маркетолог, врач, CEO)
         │
         ▼
    [React Frontend] (порт 3000)
         │
         ▼ (HTTP/HTTPS)
    [Nginx Reverse Proxy] (порт 80/443)
         │
         ▼
    [FastAPI] (порт 8000) ──────────┬──────────────► [PostgreSQL] (5432)
      │                              │                     │
      │ (JWT, CRUD, вебхуки)          ├────────────────► [Redis] (6379)
      │                              │
      ▼                              ▼
 [RabbitMQ] (5672)             [n8n] (5678)
      │                              │
      │ (очереди задач)               │ (workflows: cron, webhook)
      ▼                              │
 [Voice Bot] (8001) ◄────────────────┘
      │
      ▼ (REST API + callbacks)
 [Voximplant] ──► телефонные звонки пациентам
```

### Описание потоков

1. **Приём лида с Avito:**  
   Avito → лендинг (Flexbe) → POST /api/webhook/avito_lead → FastAPI сохраняет пациента, ставит задачу в RabbitMQ (очередь `voice_tasks`, приоритет 10).  
   Voice Bot забирает задачу, звонит в течение 5 минут.

2. **Напоминание о записи:**  
   n8n cron → GET /api/visits/pending_reminders → для каждого визита ставит задачу в RabbitMQ.  
   Voice Bot звонит за 24 часа.

3. **Реактивация RiskWhale:**  
   n8n cron (воскресенье, 10:00) → GET /api/patients?rfm_segment=RiskWhale → ставит задачи в RabbitMQ (скрипт `reactivation_risk`).  
   Voice Bot звонит, предлагает бесплатную диагностику.

4. **Follow-up после приёма:**  
   После завершения визита FastAPI через 3 дня ставит задачу в RabbitMQ (скрипт `follow_up`).  
   Voice Bot звонит и предлагает записаться на курс.

5. **Обработка пропущенного визита (no-show):**  
   n8n каждые 15 минут проверяет просроченные визиты → FastAPI обновляет статус на `no_show` и ставит задачу в RabbitMQ с задержкой 1 час.  
   Voice Bot звонит, перезаписывает.

## 3. МОДЕЛЬ ДАННЫХ (SQLAlchemy) – ПОЛНАЯ

```python
# app/models.py
from sqlalchemy import Column, Integer, String, DateTime, ForeignKey, Float, Boolean, JSON, Enum, Text
from sqlalchemy.orm import relationship
from sqlalchemy.dialects.postgresql import JSONB
from datetime import datetime
import enum

class RFMSegment(str, enum.Enum):
    ACTIVE = "active"
    RISK_WHALE = "risk_whale"       # 6-12 мес без визитов, LTV > 100k
    SLEEPING_WHALE = "sleeping_whale" # 1-2 года, LTV > 100k
    LOST_WHALE = "lost_whale"       # >2 лет, LTV > 100k
    NEW = "new"
    REGULAR = "regular"

class TargetAudience(str, enum.Enum):
    CHILD_0_3 = "child_0_3"
    CHILD_7_14 = "child_7_14"
    PREGNANT = "pregnant"
    ADULT_BACK = "adult_back"

class Patient(Base):
    __tablename__ = "patients"
    id = Column(Integer, primary_key=True)
    full_name = Column(String(200), nullable=False)
    phone = Column(String(20), unique=True, index=True, nullable=False)
    email = Column(String(100), nullable=True)
    birth_date = Column(DateTime, nullable=True)
    telegram_id = Column(String(50), nullable=True)
    consent_to_calls = Column(Boolean, default=True)
    consent_to_sms = Column(Boolean, default=True)
    
    # Маркетинговые поля
    source = Column(String(50))  # avito, prodoctors, instagram, youtube, direct
    utm = Column(JSONB, default={})
    rfm_segment = Column(Enum(RFMSegment), default=RFMSegment.NEW)
    target_audience = Column(Enum(TargetAudience), nullable=True)
    
    # Финансы и активность
    total_paid = Column(Float, default=0.0)
    last_visit_date = Column(DateTime, nullable=True)
    last_reactivation_attempt = Column(DateTime, nullable=True)
    created_at = Column(DateTime, default=datetime.utcnow)
    updated_at = Column(DateTime, default=datetime.utcnow, onupdate=datetime.utcnow)
    
    # Связи
    visits = relationship("Visit", back_populates="patient")
    courses = relationship("Course", back_populates="patient")
    calls = relationship("CallLog", back_populates="patient")
    quiz_answers = relationship("QuizAnswer", back_populates="patient")

class Doctor(Base):
    __tablename__ = "doctors"
    id = Column(Integer, primary_key=True)
    full_name = Column(String(100), nullable=False)
    speciality = Column(String(100))
    is_madyakin = Column(Boolean, default=False)
    visits = relationship("Visit", back_populates="doctor")

class Visit(Base):
    __tablename__ = "visits"
    id = Column(Integer, primary_key=True)
    patient_id = Column(Integer, ForeignKey("patients.id"))
    doctor_id = Column(Integer, ForeignKey("doctors.id"))
    service_name = Column(String(200))
    service_price = Column(Float)
    scheduled_at = Column(DateTime, nullable=False)
    status = Column(String(20), default="scheduled")  # scheduled, completed, no_show, cancelled
    was_reminded = Column(Boolean, default=False)
    follow_up_done = Column(Boolean, default=False)
    created_at = Column(DateTime, default=datetime.utcnow)
    
    patient = relationship("Patient", back_populates="visits")
    doctor = relationship("Doctor", back_populates="visits")

class Course(Base):
    __tablename__ = "courses"
    id = Column(Integer, primary_key=True)
    patient_id = Column(Integer, ForeignKey("patients.id"))
    course_type = Column(String(50))  # OPTIMA, BASE, PREGNANCY
    sessions_total = Column(Integer)
    sessions_completed = Column(Integer, default=0)
    total_amount = Column(Float)
    status = Column(String(20), default="active")  # active, completed, abandoned
    started_at = Column(DateTime)
    finished_at = Column(DateTime, nullable=True)
    patient = relationship("Patient", back_populates="courses")

class CallLog(Base):
    __tablename__ = "calls"
    id = Column(Integer, primary_key=True)
    patient_id = Column(Integer, ForeignKey("patients.id"))
    script_type = Column(String(50))   # reminder, no_show_return, reactivation_risk, follow_up, birthday
    direction = Column(String(10), default="outgoing")
    status = Column(String(20))        # answered, no_answer, busy, failed
    duration_sec = Column(Integer, default=0)
    recording_url = Column(String(255), nullable=True)
    transcript = Column(Text, nullable=True)
    intent = Column(String(50))        # agreed, rejected, rescheduled, callback_later
    created_at = Column(DateTime, default=datetime.utcnow)
    patient = relationship("Patient", back_populates="calls")

class QuizAnswer(Base):
    __tablename__ = "quiz_answers"
    id = Column(Integer, primary_key=True)
    patient_id = Column(Integer, ForeignKey("patients.id"))
    campaign_id = Column(String(50))   # например "avito_skolioz"
    question = Column(String(200))
    answer = Column(String(200))
    created_at = Column(DateTime, default=datetime.utcnow)
    patient = relationship("Patient", back_populates="quiz_answers")
```

**Шаг 1.4:** Скрипт миграции из 1С (`migration_1c.py`):

```python
import pandas as pd
from sqlalchemy import create_engine
import re

def normalize_phone(phone):
    if not phone:
        return None
    digits = re.sub(r"\D", "", str(phone))
    if len(digits) == 10:
        digits = "7" + digits
    elif len(digits) == 11 and digits[0] == "8":
        digits = "7" + digits[1:]
    if len(digits) == 11:
        return "+" + digits
    return None

def migrate():
    sync_engine = create_engine("postgresql://crm_user:strong_password@localhost:5432/crm")
    # Предполагаем, что есть CSV-файлы:
    patients_df = pd.read_csv("export_1c/clients.csv")
    patients_df['phone'] = patients_df['phone'].apply(normalize_phone)
    patients_df = patients_df.dropna(subset=['phone'])
    patients_df.to_sql("patients", sync_engine, if_exists="append", index=False)
    
    visits_df = pd.read_csv("export_1c/visits.csv")
    visits_df.to_sql("visits", sync_engine, if_exists="append", index=False)
    
    print(f"Мигрировано {len(patients_df)} пациентов, {len(visits_df)} визитов.")

if __name__ == "__main__":
    migrate()
```

### ЭТАП 3: АВТОМАТИЗАЦИЯ МАРКЕТИНГА + РЕАКТИВАЦИЯ (дни 24–33)

**Шаг 3.1:** Сервис RFM-аналитики (отдельный контейнер с cron):

```python
# rfm_service.py
import pandas as pd
from sqlalchemy import create_engine
from datetime import datetime, timedelta

engine = create_engine("postgresql://crm_user:strong_password@postgres:5432/crm")

def calculate_rfm():
    df = pd.read_sql("SELECT id, last_visit_date, total_paid FROM patients", engine)
    today = datetime.now()
    df['days_inactive'] = (today - pd.to_datetime(df['last_visit_date'])).dt.days
    df['segment'] = 'new'
    # RiskWhale: 180-365 дней неактивности и total_paid > 100000
    mask_risk = (df['days_inactive'] >= 180) & (df['days_inactive'] < 365) & (df['total_paid'] > 100000)
    df.loc[mask_risk, 'segment'] = 'risk_whale'
    # SleepingWhale: 365-730 дней и total_paid > 100000
    mask_sleep = (df['days_inactive'] >= 365) & (df['days_inactive'] < 730) & (df['total_paid'] > 100000)
    df.loc[mask_sleep, 'segment'] = 'sleeping_whale'
    # LostWhale: более 730 дней и total_paid > 100000
    mask_lost = (df['days_inactive'] >= 730) & (df['total_paid'] > 100000)
    df.loc[mask_lost, 'segment'] = 'lost_whale'
    # Обновляем БД
    for idx, row in df.iterrows():
        engine.execute(f"UPDATE patients SET rfm_segment='{row['segment']}' WHERE id={row['id']}")
    print("RFM updated")

if __name__ == "__main__":
    calculate_rfm()
```

Запускать через cron в контейнере или через n8n (HTTP Request на `/api/run_rfm`).

```python
from fastapi import Depends, HTTPException

def get_current_user(token: str = Depends(oauth2_scheme)):
    payload = jwt.decode(token, SECRET_KEY, algorithms=[ALGORITHM])
    role = payload.get("role")
    return {"id": payload["sub"], "role": role}

def require_role(required_role: str):
    def role_checker(current_user = Depends(get_current_user)):
        if current_user["role"] != required_role and current_user["role"] != "admin":
            raise HTTPException(status_code=403)
        return current_user
    return role_checker

# Использование:
```

**Шаг 5.2:** Шифрование телефонов (используем Fernet):

```python
from cryptography.fernet import Fernet
cipher = Fernet(settings.ENCRYPTION_KEY)

def encrypt_phone(phone: str) -> str:
    return cipher.encrypt(phone.encode()).decode()

def decrypt_phone(encrypted: str) -> str:
    return cipher.decrypt(encrypted.encode()).decode()
```
